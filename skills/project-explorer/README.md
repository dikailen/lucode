# 项目探索者 (Project Explorer) 使用指南

这个技能可以帮助开发者快速了解新项目的结构、技术栈、架构和部署方式。

## 快速开始

### 基本使用

当你刚接手一个新项目时，可以这样使用：

```
用户：我刚加入一个新项目，需要快速了解项目结构
技能：开始扫描项目，分析技术栈，生成项目概览报告
```

### 具体分析场景

#### 1. 项目结构分析
```
用户：请帮我分析这个项目的目录结构，解释各个目录的作用
技能：扫描并分析项目结构，提供详细的目录说明
```

#### 2. 技术栈识别
```
用户：这个项目使用了什么技术栈？前后端分别是什么？
技能：识别项目使用的编程语言、框架、库等
```

#### 3. 开发入手指导
```
用户：我想在这个项目中添加一个新的用户管理模块，应该从哪里开始？
技能：提供具体的开发步骤和建议
```

#### 4. 部署配置了解
```
用户：这个项目如何部署？需要什么环境？
技能：分析部署配置，提供部署命令和环境要求
```

## 技能功能

### 🏗️ 项目架构分析
- 理解项目的整体架构设计
- 识别架构模式（MVC、微服务、单体等）
- 分析数据流和请求处理路径

### 🛠️ 技术栈识别
- 检测编程语言和版本
- 识别前端框架（React、Vue、Angular等）

- 解释关键目录的用途
- 分析代码组织方式
- 理解模块间的依赖关系

### ⚙️ 配置文件分析
- 识别重要的配置文件
- 解释配置项的作用
- 发现环境变量设置

### 🎯 开发指导
- 建议新功能开发的起点
- 推荐代码规范和最佳实践
- 提供调试和测试建议

### 🚀 部署了解
- 提供构建和打包命令
- 解释环境配置要求
- 了解服务启动方式
- 指出访问方式和端口

## 输出格式

### 项目概览报告
技能会生成一个清晰的项目概览，包括：
- 基本信息（项目名称、类型、技术栈）
- 目录结构分析
- 核心功能模块
- 开发入手建议
- 部署和访问信息

### 技术栈详情
详细列出项目使用的技术：
- 前端技术（框架、状态管理、构建工具）
- 后端技术（框架、数据库、缓存）
- 开发工具（包管理、测试框架、代码规范）

### 开发建议
提供具体的开发指导：
- 新功能开发的步骤
- 代码规范和最佳实践
- 注意事项和常见问题

## 使用示例

### 示例1：React项目分析
```
用户：这是一个React项目，请帮我快速了解项目结构
技能：
✅ 项目类型：JavaScript/Node.js前端应用
✅ 主要技术：React, TypeScript, Webpack
✅ 目录结构：
  - src/components/ React组件
  - src/pages/ 页面组件
  - src/utils/ 工具函数
  - public/ 静态资源
🎯 开发建议：在src/components/下创建新的React组件
🚀 部署：npm run dev 启动开发服务器，访问 http://localhost:3000
```

### 示例2：Spring Boot项目分析
```
用户：这是一个Java项目，我想了解如何添加新功能
技能：
✅ 项目类型：Java Spring Boot后端应用
✅ 主要技术：Spring Boot, Maven, MySQL
✅ 目录结构：
  - src/main/java/ Java源码
  - src/main/resources/ 配置文件
  - src/test/ 测试代码
🎯 开发建议：在src/main/java/下创建新的Controller和Service类
🚀 部署：mvn spring-boot:run 启动应用，访问 http://localhost:8080
```

## 注意事项

1. **项目访问**：技能需要能够访问项目文件，确保项目路径正确
2. **只读模式**：技能只进行读取和分析，不会修改任何项目文件
3. **依赖文件**：确保项目的配置文件（package.json、pom.xml等）是最新的
4. **复杂项目**：对于大型复杂项目，可能需要多次分析才能完全理解

## 常见问题

**Q: 这个技能能处理哪些类型的项目？**
A: 目前支持JavaScript/TypeScript、Python、Java、Go、Rust等主流技术栈的项目。

**Q: 技能会修改项目文件吗？**
A: 不会，技能只进行读取和分析，不会修改任何项目文件。

**Q: 如何获得最准确的分析结果？**
A: 确保项目的README、package.json等配置文件是最新的，这样能提供更准确的信息。

**Q: 技能能帮助调试项目吗？**
A: 虽然主要专注于项目理解，但也会提供一些基本的调试建议和常见问题解决方案。
